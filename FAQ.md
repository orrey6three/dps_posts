# FAQ - Часто задаваемые вопросы

## Общие вопросы

### Что такое DPS45?

DPS45 - это веб-приложение для отслеживания постов ДПС в городе Шумиха, Курганская область. Пользователи могут отмечать посты как актуальные или неактуальные, помогая другим водителям.

### Нужна ли регистрация?

Нет! Приложение работает без регистрации. Устройство идентифицируется автоматически через уникальный ID в localStorage.

### Как часто можно голосовать?

Один раз в 10 минут для каждого поста. Это защита от спама.

### Что означают цвета маркеров?

- 🔴 **Красный**: последний голос "Актуально" (пост активен)
- ⚫ **Серый**: последний голос "Неактуально" (поста нет)
- ⚪ **Бледно-серый**: нет активности более 6 часов (данные устарели)

### Как работает система голосования?

1. Каждое устройство получает уникальный ID
2. При голосовании проверяется, голосовали ли вы за последние 10 минут
3. Если нет - голос принимается и сохраняется в базе
4. Статистика обновляется для всех пользователей

## Технические вопросы

### На каких устройствах работает?

- ✅ iOS (Safari, Chrome)
- ✅ Android (Chrome, Firefox)
- ✅ Десктоп (все современные браузеры)

Оптимизировано для мобильных устройств!

### Какие данные хранятся на моем устройстве?

Только `device_id` в localStorage - случайный UUID для идентификации устройства. Никаких личных данных!

### Что если я очищу кеш браузера?

Будет сгенерирован новый `device_id`, и вы сможете голосовать снова. Старые голоса останутся в базе.

### Работает ли оффлайн?

Нет, требуется интернет-соединение для:
- Загрузки карты
- Отправки голосов
- Получения обновлений

### Насколько точная карта?

Используется OpenStreetMap - бесплатная карта с достаточной точностью для городского масштаба. Координаты постов настраиваются администратором.

## Вопросы администраторов

### Как получить доступ к админке?

1. Перейдите на `/admin`
2. Введите пароль из переменной окружения `ADMIN_PASSWORD`
3. Получите доступ на 24 часа

### Как добавить новый пост?

1. Войдите в админку
2. Нажмите "+ Добавить пост"
3. Заполните форму (название, адрес, координаты)
4. Сохраните

### Где взять координаты?

- Google Maps: правый клик → координаты
- Яндекс.Карты: правый клик → "Что здесь?"
- OpenStreetMap: в URL после поиска

### Можно ли удалить голоса?

Да, через Supabase SQL Editor:

```sql
-- Удалить все голоса для поста
DELETE FROM votes WHERE post_id = 'uuid-поста';

-- Удалить все голоса
DELETE FROM votes;
```

### Как посмотреть статистику?

В админке отображается:
- Количество голосов "Актуально"
- Количество голосов "Неактуально"
- Общее количество голосов

Для детальной статистики используйте Supabase Dashboard.

### Можно ли заблокировать пользователя?

Нет встроенного механизма блокировки. Можно вручную удалять голоса от конкретного `device_id` в Supabase.

## Вопросы безопасности

### Безопасна ли база данных?

Да! Используется Supabase с:
- Row Level Security (RLS)
- Защитой от SQL injection
- Анонимным ключом (без прямого доступа к данным)

### Можно ли взломать голосование?

Базовая защита включает:
- ✅ Ограничение по времени (10 минут)
- ✅ Проверку на бэкенде
- ✅ Уникальный device_id

Однако продвинутый пользователь может:
- Создать несколько device_id (очистив localStorage)
- Использовать разные устройства

Для полной защиты требуется регистрация и капча (не реализовано).

### Видны ли мои данные другим пользователям?

Другие пользователи видят только:
- Время последних голосов
- Количество голосов

НЕ видят:
- Ваш device_id
- IP-адрес
- Личную информацию

### Что с конфиденциальностью?

Приложение не собирает:
- ❌ Имена
- ❌ Email
- ❌ Номера телефонов
- ❌ Геолокацию (кроме при добавлении постов администратором)

Собирается только:
- ✅ Device ID (анонимный)
- ✅ Голоса (пост + тип + время)

## Проблемы и ошибки

### Карта не загружается

**Возможные причины:**
1. Нет интернета
2. Заблокирован OpenStreetMap
3. Проблема с Leaflet CDN

**Решение:**
- Проверьте соединение
- Попробуйте другую сеть
- Очистите кеш браузера

### Кнопки голосования не работают

**Возможные причины:**
1. Вы уже голосовали (10 минут не прошло)
2. Проблема с API
3. Supabase не отвечает

**Решение:**
- Подождите 10 минут
- Проверьте Console (F12) на ошибки
- Обновите страницу

### Показывает "Неверный пароль" в админке

**Решение:**
- Проверьте переменную `ADMIN_PASSWORD` в .env
- Убедитесь, что нет лишних пробелов
- После изменения .env перезапустите сервер

### Маркеры отображаются не в тех местах

**Решение:**
1. Зайдите в админку
2. Найдите пост с неправильными координатами
3. Нажмите "Редактировать"
4. Исправьте широту/долготу
5. Сохраните

### Ошибка "Missing Supabase credentials"

**Решение:**
- Создайте файл `.env` из `.env.example`
- Заполните `SUPABASE_URL` и `SUPABASE_ANON_KEY`
- Перезапустите приложение

## Вопросы разработки

### Можно ли изменить дизайн?

Да! Редактируйте:
- `public/css/style.css` - стили
- `public/index.html` - HTML структура
- Tailwind классы прямо в HTML

### Как добавить новую функцию?

1. Обновите API (`routes/api.js` или `routes/admin.js`)
2. Обновите frontend (`public/js/app.js` или `admin.js`)
3. При необходимости обновите базу данных (migration.sql)
4. Протестируйте локально
5. Commit → Push → Vercel автоматически задеплоит

### Можно ли использовать другую карту?

Да! Замените Leaflet на:
- Google Maps (платно)
- Yandex Maps
- Mapbox (частично платно)
- 2GIS

Нужно будет обновить `public/js/app.js`.

### Как добавить push-уведомления?

Требуется:
1. Service Worker
2. Push API
3. Backend для отправки уведомлений (Firebase, OneSignal)

Это выходит за рамки текущего MVP.

### Можно ли добавить регистрацию?

Да! Используйте Supabase Auth:
1. Включите Email Auth в Supabase
2. Добавьте компоненты регистрации/входа
3. Замените device_id на user_id
4. Обновите Row Level Security политики

### Как сделать приложение быстрее?

Оптимизации:
- ✅ Добавить кеширование API ответов
- ✅ Использовать CDN для статических файлов
- ✅ Оптимизировать SQL запросы
- ✅ Добавить Service Worker для оффлайн режима
- ✅ Минифицировать JS/CSS

## Вопросы масштабирования

### Сколько пользователей выдержит?

На бесплатном плане:
- **Vercel**: 100GB bandwidth/мес (~100,000 посетителей)
- **Supabase**: 500MB база + 1GB storage + 2GB transfer

Для города Шумиха этого более чем достаточно!

### Что если нужно больше?

Переходите на платные планы:
- **Vercel Pro**: $20/мес (больше bandwidth)
- **Supabase Pro**: $25/мес (больше базы и запросов)

### Можно ли использовать для другого города?

Да! Просто измените:
1. Координаты центра карты в `app.js`
2. Посты в `migration.sql`
3. Название проекта

Можно даже развернуть несколько копий для разных городов!
